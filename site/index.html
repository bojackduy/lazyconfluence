Terminal-first Confluence reader

Browse Confluence docs and image-heavy pages in your terminal.

Why it exists

A focused reader, not another browser tab.

Confluence is useful, but jumping between browser tabs breaks terminal flow. lazyconfluence keeps the read path local-first after sync, so browsing a page tree, scanning content, and searching docs feels fast and keyboard-native.

Features

Built around terminal document work, including images.

Local-first after sync

Remote Confluence requests happen only when you explicitly run sync. Normal reading uses your local SQLite index.

Space and page navigation

Start from one active space, move through the page tree, and switch spaces without losing keyboard flow.

Progressive image rendering

Cached Confluence PNG attachments render as terminal previews without native image protocols; unsupported media and terminals without preview support keep labeled placeholders.

Search and relationships

Search synced pages, inspect child pages, outgoing links, backlinks, and page outlines from the same interface.

Draft locally

Create local drafts, preview differences, stage changes, and keep remote write-back explicit.

Safe demo mode

Run with synthetic data for screenshots, recordings, demos, and bug reports without exposing company docs.

Quick start

Install, configure, sync, read.

  1. Install Bun. curl -fsSL https://bun.sh/install | bash
  2. Install lazyconfluence. npm install -g @bojackduy/lazyconfluence
  3. Configure Atlassian credentials. lazyconfluence init
  4. Sync configured spaces. lazyconfluence sync
  5. Open the terminal reader. lazyconfluence

Privacy model

No surprise remote reads while browsing.

Your Atlassian token is stored in your local user config directory. Synced content stays in your local index, and the TUI reads that index instead of calling Confluence while you browse.

lazyconfluence is open source on GitHub and published on npm.
